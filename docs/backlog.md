# 1. Overview

Dokumen ini berisi seluruh daftar pekerjaan (Product Backlog) yang akan dikerjakan selama pengembangan Frontend Library Management System.

Backlog disusun berdasarkan prioritas agar AI Agent dapat mengembangkan aplikasi secara bertahap sesuai Sprint Plan.

Setiap backlog memiliki:

- ID
- Feature
- Priority
- Sprint
- Status
- Description
- Acceptance Criteria

# 4. Definition of Ready (DoR)

Suatu backlog dapat dikerjakan apabila:

- Requirement telah terdokumentasi.
- UI telah didefinisikan pada Design System.
- Halaman telah dijelaskan pada Page Specification.
- Sprint telah ditentukan.
- Tidak ada dependency yang menghambat.

---

# 5. Definition of Done (DoD)

Sebuah backlog dianggap selesai apabila:

- Implementasi selesai.
- Tidak ada error TypeScript.
- Tidak ada error ESLint.
- Build berhasil.
- Responsive.
- Menggunakan reusable component.
- Mengikuti Coding Guidelines.
- Mengikuti Design System.
- Status pada `project_state.md` diperbarui.

---

# 6. AI Agent Rules

AI Agent wajib:

1. Mengambil pekerjaan berdasarkan prioritas.
2. Mengikuti urutan Sprint.
3. Menyelesaikan backlog pada Sprint aktif sebelum berpindah ke Sprint berikutnya.
4. Tidak mengubah prioritas tanpa persetujuan.
5. Memperbarui `project_state.md` setelah backlog pada Sprint selesai.
