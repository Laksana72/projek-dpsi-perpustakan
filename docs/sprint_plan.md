# Sprint Overview

# Sprint Workflow

Untuk setiap Sprint, AI Agent wajib:

1. Membaca project_state.md.
2. Mengidentifikasi Sprint aktif.
3. Membaca dokumen yang relevan.
4. Mengimplementasikan fitur sesuai Sprint.
5. Melakukan pengecekan build.
6. Memperbarui project_state.md.
7. Melanjutkan ke Sprint berikutnya setelah Sprint saat ini selesai.

---

# Definition of Done (DoD)

Sebuah Sprint dianggap selesai apabila:

- Semua deliverables selesai.
- Tidak ada error TypeScript.
- Tidak ada error ESLint.
- Build berhasil.
- UI sesuai Design System.
- Responsive.
- Menggunakan reusable component.
- Mengikuti Coding Guidelines.
- project_state.md telah diperbarui.

---

# AI Agent Rules

AI Agent wajib:

- Mengikuti urutan Sprint.
- Tidak mengerjakan Sprint berikutnya sebelum Sprint aktif selesai.
- Tidak mengubah struktur proyek tanpa pembaruan dokumentasi.
- Mengikuti seluruh dokumen dalam folder docs/.
- Memperbarui project_state.md setiap Sprint selesai.
