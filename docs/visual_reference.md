# Visual Reference
Version : 1.0

Project

UAD Library Management System

Platform

Web Application

Purpose

Dokumen ini menjelaskan karakter visual aplikasi.

AI wajib mengikuti dokumen ini untuk menghasilkan tampilan yang semirip mungkin dengan desain referensi.

Target visual minimal 95% sesuai desain.

---

# Overall Impression

Website harus memberikan kesan.

Modern

Professional

Academic

Elegant

Minimalist

Clean

Comfortable

Friendly

Website tidak boleh terlihat seperti template dashboard gratis.

Gunakan banyak white space.

Gunakan border radius besar.

Gunakan shadow lembut.

Gunakan transisi halus.

---

# Visual Style

Menggunakan gaya modern.

Dominasi warna biru Universitas Ahmad Dahlan.

Background terang.

Card putih.

Hero menggunakan foto kampus.

Footer berwarna biru tua.

---

# Hero Visual

Hero merupakan fokus utama Landing Page.

Hero memenuhi bagian atas layar.

Background menggunakan foto Gedung Universitas Ahmad Dahlan.

Foto tetap terlihat jelas.

Jangan terlalu gelap.

Tambahkan overlay gradient transparan agar teks mudah dibaca.

Background tidak boleh blur.

Background harus memenuhi seluruh Hero.

Background Position

Center

Background Size

Cover

---

# White Space

Gunakan white space yang luas.

Jarak antar section besar.

Jangan membuat halaman terlihat padat.

Semua komponen harus memiliki ruang bernapas.

---

# Visual Hierarchy

Prioritas visual.

Hero

↓

Search

↓

Feature

↓

Keunggulan

↓

CTA

↓

Footer

Heading Hero merupakan elemen paling dominan.

Search menjadi fokus kedua.

Feature menjadi fokus ketiga.

---

# Color Mood

Gunakan kombinasi.

Dark Blue

White

Light Gray

Primary Blue

Warna tidak boleh terlalu mencolok.

---

# Icon Style

Gunakan Lucide React.

Outline Icon.

Ukuran konsisten.

Icon menggunakan warna Primary.

---

# Image Style

Gunakan gambar berkualitas tinggi.

Resolusi tinggi.

Tidak pecah.

Tidak menggunakan watermark.

Cover buku memiliki rasio yang konsisten.

---

# Card Style

Background putih.

Radius besar.

Shadow tipis.

Hover naik sedikit.

Tidak menggunakan border tebal.

---

# Typography Style

Gunakan font Poppins.

Heading tegas.

Body mudah dibaca.

Gunakan line-height yang nyaman.

Jangan menggunakan lebih dari empat tingkat font weight.

---

# Button Style

Rounded.

Modern.

Tidak terlalu besar.

Hover halus.

Primary Button menggunakan warna biru.

Secondary Button menggunakan outline.

---

# Responsive Principle

Desktop menjadi prioritas.

Tablet tetap nyaman.

Mobile sederhana.

Layout tidak boleh rusak.

Tidak boleh muncul horizontal scroll.

---

# Dashboard Style

Dashboard User dan Dashboard Admin menggunakan layout berbeda dengan Landing Page.

Landing Page tidak memiliki Sidebar.

Dashboard menggunakan Sidebar.

Dashboard memiliki Header.

Landing Page menggunakan Footer.

Dashboard tidak menggunakan Footer.

---

# Consistency Rules

Semua halaman harus menggunakan:

Typography yang sama.

Color Palette yang sama.

Spacing yang sama.

Border Radius yang sama.

Shadow yang sama.

Animation yang sama.

---

AI tidak boleh mengubah gaya visual.

AI tidak boleh membuat variasi layout.
